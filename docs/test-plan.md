# План и Методика Испытаний (ПМИ) — FlowDesk

> Программа и методика испытаний системы автоматизации гибких рабочих мест

---

## 1. Общие сведения

| Параметр | Значение |
|----------|----------|
| **Объект испытаний** | FlowDesk — REST API бэкенд-сервер |
| **Цель испытаний** | Проверка соответствия реализации функциональным и нефункциональным требованиям ТЗ |
| **Вид испытаний** | Функциональное тестирование, тестирование безопасности, нагрузочное тестирование |
| **Среда испытаний** | Docker Compose (PostgreSQL 16-alpine + Go API Server + migrate) |
| **Инструменты** | cURL / Postman / httpie, pgAdmin / psql, k6 (нагрузочное) |

---

## 2. Подготовка к испытаниям

### 2.1 Развёртывание среды

```bash
# 1. Клонировать репозиторий
git clone <repo-url> && cd office-space-allocation

# 2. Настроить переменные окружения
cp .env.example .env

# 3. Запустить инфраструктуру
docker-compose up -d

# 4. Убедиться, что все сервисы запущены
docker-compose ps
# postgres — healthy
# migrate — exited (0)
# api — running, порт 8080
```

### 2.2 Начальные данные

Перед началом испытаний необходимо убедиться, что:
- Миграции БД применены успешно (8 миграций)
- В системе существует хотя бы 1 этаж, 1 зона и 3 рабочих места
- Создан тестовый пользователь через `POST /auth/register`

### 3.5 Тестирование безопасности

#### ТС-SEC-01: Проверка JWT-аутентификации

| # | Действие | Ожидаемый результат |
|---|---------|---------------------|
| 1 | `POST /reservations` без `Authorization` | `401 Unauthorized` |
| 2 | `POST /reservations` с невалидным JWT | `401 Unauthorized` |
| 3 | `DELETE /reservations/{id}` без `Authorization` | `401 Unauthorized` |
| 4 | `POST /reservations/{id}/release` без `Authorization` | `401 Unauthorized` |
| 5 | `GET /reservations` без `Authorization` | `401 Unauthorized` |

---

#### ТС-SEC-02: Проверка авторизации (role-based)

| # | Действие | Ожидаемый результат |
|---|---------|---------------------|
| 1 | Member пытается отменить чужую бронь | `403 Forbidden` |
| 2 | Admin отменяет чужую бронь | `204 No Content` (успех) |
| 3 | Member пытается досрочно освободить чужую бронь | `403 Forbidden` |
| 4 | Admin досрочно освобождает чужую бронь | `200 OK` (успех) |

---

#### ТС-SEC-03: Проверка целостности данных

| # | Действие | Ожидаемый результат |
|---|---------|---------------------|
| 1 | Попытка бронирования с `from ≥ to` | `400 Bad Request` (CHECK constraint) |
| 2 | Два параллельных бронирования одного места | Одно — `201`, второе — `409` (EXCLUDE constraint) |
| 3 | Отменённая бронь НЕ должна иметь `cancelledAt = NULL` | `cancelled_at IS NOT NULL` для status = cancelled |

## 5. Критерии завершения испытаний

| Критерий | Условие прохождения |
