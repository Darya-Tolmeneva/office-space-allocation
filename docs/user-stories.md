# User Stories (US) — FlowDesk

> Пользовательские истории системы автоматизации гибких рабочих мест

---

## Роли

| Роль | Описание |
|------|----------|
| **Сотрудник** | Рядовой пользователь системы (role = `member`) |
| **Администратор** | Управляющий каталогом и бронированиями (role = `admin`) |

---

## Аутентификация

### US-AUTH-01: Регистрация в системе
**Как** новый сотрудник,  
**я хочу** зарегистрироваться, указав email, пароль (≥ 8 символов) и ФИО,  
**чтобы** получить доступ к системе бронирования рабочих мест.

**Критерии приёмки:**
- При успешной регистрации возвращаются данные пользователя и пара токенов (access + refresh)
- При дублировании email возвращается ошибка `409 Conflict`
- При невалидных данных возвращается ошибка `400 Bad Request`

---

### US-AUTH-02: Вход в систему
**Как** зарегистрированный сотрудник,  
**я хочу** войти по email и паролю,  
**чтобы** получить токен доступа для работы с системой.

**Критерии приёмки:**
- При успешном входе возвращается `AuthResponse` с парой токенов
- При неверных учётных данных — `401 Unauthorized`

---

### US-AUTH-03: Обновление токена доступа
**Как** аутентифицированный пользователь,  
**я хочу** обновить свой access-токен по refresh-токену,  
**чтобы** продолжить работу без повторного ввода пароля.

**Критерии приёмки:**
- Возвращается новая пара токенов
- Просроченный refresh-токен → `401 Unauthorized`

---

## Каталог рабочих мест

### US-CAT-01: Просмотр списка этажей
**Как** сотрудник,  
**я хочу** видеть список всех этажей офиса,  
**чтобы** выбрать, на каком этаже искать место.

**Критерии приёмки:**
- Отображаются: `id`, `name`, `timezone`, `floorPlanUrl`
- Список возвращается без аутентификации

---

### US-CAT-02: Просмотр плана этажа
**Как** сотрудник,  
**я хочу** видеть план конкретного этажа со всеми рабочими местами,  
**чтобы** выбрать удобное место визуально.

**Критерии приёмки:**
- Возвращается информация об этаже, список зон и массив рабочих мест с координатами (`x`, `y`)
- Каждое место содержит: `label`, `status`, `features`, `position`

---

### US-CAT-03: Фильтрация рабочих мест
**Как** сотрудник,  
**я хочу** искать рабочие места по фильтрам: этаж, зона, характеристики (монитор, тишина, стоячее место и др.),  
**чтобы** быстро найти подходящее место.

**Критерии приёмки:**
- Поддерживаются query-параметры: `floorId`, `zoneId`, `features`
- `features` передаётся через запятую: `monitor,quiet,standing`
- Возвращаются только места, соответствующие всем указанным фильтрам

---

### US-CAT-04: Проверка доступности места
**Как** сотрудник,  
**я хочу** проверить, свободно ли конкретное место в нужный мне временной интервал,  
**чтобы** убедиться перед бронированием.

**Критерии приёмки:**
- Указываются `from` и `to` (обязательные, формат date-time)
- Возвращается массив слотов со статусами: `available`, `reserved`, `occupied`, `disabled`
- Для занятых слотов указывается `reservationId`

---

## Бронирование

### US-RES-01: Ручное бронирование места
**Как** сотрудник,  
**я хочу** забронировать конкретное рабочее место на заданный временной интервал,  
**чтобы** гарантировать себе место на нужное время.

**Критерии приёмки:**
- Указываются: `deskId`, `from`, `to`, опционально `note`
- Имя владельца (`holderName`) заполняется автоматически из профиля
- При пересечении с существующей бронью — `409 Conflict`
- Место в статусе `disabled` не бронируется — `409 Conflict`
- Требуется аутентификация (JWT)

---

### US-RES-02: Автоматический подбор места
**Как** сотрудник,  
**я хочу** нажать одну кнопку и получить подходящее место автоматически,  
**чтобы** не тратить время на ручной поиск.

**Критерии приёмки:**
- Указываются: `from`, `to`, опционально `floorId`, `zoneId`, `requiredFeatures`
- Система выбирает первое свободное `active` место по `label ASC`, соответствующее фильтрам
- Если подходящих мест нет — `404 Not Found`
- Источник бронирования = `auto`

---

### US-RES-03: Превью авто-подбора
**Как** сотрудник,  
**я хочу** посмотреть, какое место предложит система, **прежде чем** бронировать,  
**чтобы** принять осознанное решение.

**Критерии приёмки:**
- Возвращается предложенное место (`Desk`), оценка (`score` 0–1) и список причин выбора
- Бронирование **не создаётся**

---

### US-RES-04: Просмотр своих бронирований
**Как** сотрудник,  
**я хочу** видеть список своих бронирований с возможностью фильтрации,  
**чтобы** отслеживать текущие и прошлые брони.

**Критерии приёмки:**
- Поддерживаются фильтры: `deskId`, `floorId`, `from`, `to`, `status` (active / cancelled / completed / all)
- По умолчанию `status = active`
- Требуется аутентификация

---

### US-RES-05: Отмена бронирования
**Как** сотрудник,  
**я хочу** отменить своё бронирование,  
**чтобы** освободить место для других.

**Критерии приёмки:**
- Бронирование переходит в статус `cancelled`, фиксируется `cancelledAt`
- Место становится доступным для бронирования другими
- Отмена чужой брони → `403 Forbidden` (кроме администраторов)

---

### US-RES-06: Досрочное освобождение места
**Как** сотрудник,  
**я хочу** освободить забронированное место раньше времени, указав причину,  
**чтобы** другие коллеги могли им воспользоваться.

**Критерии приёмки:**
- Бронирование переходит в статус `completed`, фиксируются `releasedAt` и `releaseReason`
- Причина — опциональное текстовое поле
- Освобождение чужой брони → `403 Forbidden` (кроме администраторов)

---

### US-RES-07: Управление чужими бронированиями (администратор)
**Как** администратор,  
**я хочу** отменять или досрочно освобождать бронирования любых сотрудников,  
**чтобы** управлять загрузкой офиса в исключительных ситуациях.

**Критерии приёмки:**
- Администратор (role = `admin`) может выполнять `DELETE` и `release` для любого `reservationId`
- Обычный сотрудник — только для своих бронирований

---

## Аналитика

### US-ANA-01: Просмотр сводной аналитики
**Как** администратор / офис-менеджер,  
**я хочу** видеть дашборд с ключевыми метриками загрузки офиса,  
**чтобы** принимать решения об оптимизации пространства.

**Критерии приёмки:**
- Отображаются метрики:
  - Средняя загрузка (`averageOccupancy`, 0–1)
  - Пиковый день (`peakDay`, напр. `Tuesday`)
  - Пиковая загрузка (`peakOccupancy`)
  - Доля авто-бронирований (`autoPickRatio`)
  - Общее число бронирований (`totalReservations`)
  - Досрочные освобождения (`earlyReleases`)
  - Самая популярная зона (`topZone: { zoneId, name }`)
- Поддерживается фильтрация по `floorId`, `from`, `to`
